dynamic programing  - DP


dp- technique,breaking into smaller, sub-problems.used for performance
when we need optimal choice(when program repeating calculations)
Overlapping Sub-problems - using cache/table
Optimal Substructure- calculating small things,will calculate big thing.(use formula/transition loop)


top-down(memoization)  - you start at top big problem and break it down recursively. (recursion+cache dict)
using call stack + cache.(call stack - running function and where it saved / cache where answers are saved)
memory O(n)
minus - in python recursion has limit, if n is so big like 20000, top down crash program. (`RecursionError: maximum recursion depth exceeded`.) for not crash use Bottom-Up (Tabulation) it crash because for fib(20000) if 20000 call stack and it make server slower.and tabulation's time there is one running loop.
use when we need all subproblems.



Bottom-Up (Tabulation) - start from bottom.avoid recursions and use loops,fill table/array stepbystep.
memory O(n) or O(n^2) depend on problem.
speed fast.use when speed is critical and we need all subproblems.
